Guia Prático: Git & GitHub

Conceitos essenciais, fluxos de trabalho e resolução de conflitos

Conceitos de Git & GitHub

  • Git: sistema de controle de versão distribuído.
  • Repositório: pasta com histórico versionado.
  • Commit: foto do seu código em um momento (mensagem clara!).
  • Remote: repositório hospedado (ex.: GitHub).
  • git clone, git add, git commit, git push, git pull.

Fluxos de Trabalho (Git/GitHub)

GitHub Flow: main estável → criar branch de feature → PR → review → merge → deploy.

Nosso fluxo neste projeto:

  1. Branches principais: main e develop.
  2. Features em feat/alguma-coisa → PR para develop.
  3. Após 3 PRs aceitos: PR de develop → main → deploy.

Como Resolver Conflitos

  1. Atualize sua branch: git fetch origin e git merge origin/develop (ou rebase).
  2. Abra os arquivos marcados com conflitos <<<<<<< / ======= / >>>>>>>.
  3. Edite escolhendo o conteúdo final e salve.
  4. Marque como resolvido: git add.
  5. Finalize: git commit (merge) ou git rebase --continue (rebase).

Dica: prefira mensagens de commit claras e pequenas mudanças por PR.
